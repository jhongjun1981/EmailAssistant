# EmailAssistant 一键部署安装说明

## 方式一：Windows 本机安装

### 第一步：安装
双击 `install.bat`，自动完成：
- 检查 Python 环境（需 3.10+）
- 安装所有依赖
- 生成配置文件

### 第二步：配置
打开 `config.json`，填入你的 LLM API Key（至少填一个）：
- 智谱 GLM（免费）：去 https://open.bigmodel.cn 申请
- DeepSeek / 通义千问 / Claude：按需填写
- 不想配 Key？把 `"model"` 改为 `"local"` 即可使用本地规则

### 第三步：启动
双击 `start_all.bat`，自动启动 API 服务 + GUI 界面。
关闭 GUI 窗口时 API 服务会自动停止。

---

## 方式二：Docker 部署

### 第一步：准备配置
```bash
cp config.example.json config.json
```
编辑 `config.json` 填入 API Key。

### 第二步：启动
```bash
docker-compose up -d
```

### 第三步（可选）：启用 MCP SSE
```bash
docker-compose --profile mcp-sse up -d
```

---

## 前置条件

- **EmailMarketer** 服务需要运行在 `localhost:8100`（负责实际发送邮件）
- Windows 安装需要 Python 3.10+
- Docker 部署需要 Docker 和 Docker Compose

## 端口说明

| 服务 | 端口 | 说明 |
|------|------|------|
| API | 8200 | FastAPI 接口 |
| MCP SSE | 8201 | MCP SSE 传输（可选） |

## MCP 集成（Claude Code / Claude Desktop）

在项目的 `.mcp.json` 中添加以下配置，重启后即可使用：

```json
"email-assistant": {
  "command": "python",
  "args": ["<你的路径>/EmailAssistant/run_mcp.py"],
  "cwd": "<你的路径>/EmailAssistant",
  "env": {
    "EA_API_URL": "http://localhost:8200"
  }
}
```

添加后会获得 4 个 MCP 工具：
- `smart_email` — 自然语言发邮件
- `parse_email_intent` — 解析意图
- `send_email` — 直接发邮件
- `email_chat` — 智能聊天

## 常见问题

**Q: 启动报错 "未检测到 Python"？**
A: 安装 Python 3.10+ 并确保添加到系统 PATH。

**Q: 发邮件失败？**
A: 确认 EmailMarketer 服务已启动（`localhost:8100`）。

**Q: 不想用 LLM，能离线使用吗？**
A: 可以。将 `config.json` 中的 `"model"` 改为 `"local"`，使用本地规则解析。
